## Overview

The Solana Bank Asset Tokenization Platform is designed as a modular, microservices-based architecture that leverages established infrastructure providers for compliance, custody, and tokenization while focusing development effort on the user experience and orchestration layer.

## Design Principles

1. **Integration-First**: Leverage established providers (Securitize, Civic, Anchorage) instead of building from scratch
2. **Separation of Concerns**: Clear boundaries between client, API, blockchain, and infrastructure layers
3. **Compliance by Design**: KYC/AML checks embedded at every critical touchpoint
4. **Horizontal Scalability**: Stateless services that can scale independently
5. **Audit Trail**: Complete on-chain and off-chain transaction logging

#### Asset Service
Handles all asset-related operations.

```typescript
interface AssetService {
  // CRUD Operations
  createAsset(data: AssetCreateDTO): Promise<Asset>;
  getAsset(id: string): Promise<Asset>;
  updateAsset(id: string, data: AssetUpdateDTO): Promise<Asset>;
  deleteAsset(id: string): Promise<void>;
  
  // Listing Operations
  listAssets(filters: AssetFilters): Promise<PaginatedAssets>;
  searchAssets(query: string): Promise<Asset[]>;
  
  // Tokenization
  initiateTokenization(assetId: string): Promise<TokenizationJob>;
```

Manages user authentication and profiles.

```typescript
interface UserService {
```

```rust
    pub authority: Pubkey,           // Bank's authority
    pub mint: Pubkey,                // Token mint address
    pub metadata_uri: String,        // IPFS/Arweave URI
    pub asset_type: AssetType,       // Real estate, equipment, etc.

    pub total_supply: u64,           // Total tokens minted

    pub created_at: i64,             // Unix timestamp
    pub updated_at: i64,             // Unix timestamp
    pub bump: u8,                    // PDA bump
}

#[account]

    pub buyer: Pubkey,               // Buyer's address
    pub seller: Pubkey,              // Seller's address
    pub asset_mint: Pubkey,          // Asset token mint
    pub payment_mint: Pubkey,        // Payment token (USDC)
    pub asset_amount: u64,           // Tokens to transfer
    pub payment_amount: u64,         // Payment amount
    pub status: EscrowStatus,        // Pending, released, refunded
    pub created_at: i64,             // Unix timestamp
    pub expires_at: i64,             // Expiration timestamp
    pub bump: u8,                    // PDA bump
}

// Auction Engine
#[account]
pub struct Auction {
    pub authority: Pubkey,           // Auction creator

    pub payment_mint: Pubkey,        // Payment token
    pub reserve_price: u64,          // Minimum price
    pub current_bid: u64,            // Highest bid
    pub current_bidder: Option<Pubkey>, // Highest bidder
    pub start_time: i64,             // Auction start
    pub end_time: i64,               // Auction end
    pub status: AuctionStatus,       // Active, settled, cancelled
    pub bump: u8,                    // PDA bump
}
```

## Data Models

### Database Schema (PostgreSQL)

```sql
-- Users
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255),
    role VARCHAR(50) NOT NULL,  -- 'bank_admin', 'investor', 'platform_admin'
    wallet_address VARCHAR(44),
    civic_pass_token VARCHAR(255),
    kyc_status VARCHAR(50) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Banks
CREATE TABLE banks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    registration_number VARCHAR(100),
    jurisdiction VARCHAR(100),
    securitize_issuer_id VARCHAR(255),
    status VARCHAR(50) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Assets
CREATE TABLE assets (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    bank_id UUID REFERENCES banks(id),
    name VARCHAR(255) NOT NULL,
    description TEXT,
    asset_type VARCHAR(100) NOT NULL,
    total_value DECIMAL(18,2) NOT NULL,
    mint_address VARCHAR(44),
    metadata_uri VARCHAR(255),
    status VARCHAR(50) DEFAULT 'draft',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Documents
CREATE TABLE documents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    asset_id UUID REFERENCES assets(id),
    type VARCHAR(100) NOT NULL,
    name VARCHAR(255) NOT NULL,
    s3_key VARCHAR(255) NOT NULL,
    ipfs_hash VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Transactions
CREATE TABLE transactions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    asset_id UUID REFERENCES assets(id),
    buyer_id UUID REFERENCES users(id),
    seller_id UUID REFERENCES users(id),
    transaction_type VARCHAR(50) NOT NULL,
    amount DECIMAL(18,2) NOT NULL,
    token_amount DECIMAL(18,8) NOT NULL,
    tx_signature VARCHAR(88),
    status VARCHAR(50) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Audit Logs
CREATE TABLE audit_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    action VARCHAR(100) NOT NULL,
    entity_type VARCHAR(100) NOT NULL,
    entity_id UUID NOT NULL,
    old_value JSONB,
    new_value JSONB,
    ip_address INET,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## Disaster Recovery

### Backup Strategy

| Component | Backup Frequency | Retention | Recovery Time |
|-----------|------------------|-----------|---------------|
| PostgreSQL | Continuous + Daily Snapshots | 30 days | < 1 hour |
| Redis | Hourly Snapshots | 7 days | < 15 minutes |
| S3 Documents | Cross-region Replication | Indefinite | Real-time |
| Blockchain State | On-chain (immutable) | Permanent | N/A |

### Failover Strategy

- **Multi-AZ Deployment**: All services deployed across multiple availability zones
- **Database Failover**: Automated RDS failover with Multi-AZ configuration
- **Cross-Region DR**: Secondary region with read replicas for critical data
- **Blockchain Redundancy**: Multiple RPC endpoints with automatic failover
